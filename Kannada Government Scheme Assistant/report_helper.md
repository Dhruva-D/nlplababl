## 1. Abstract

Access to government welfare schemes remains a significant challenge for rural and semi-urban citizens, primarily due to language barriers and the complexity of navigating official portals. This project presents the **Kannada Government Scheme Assistant**, an AI-powered full-stack web application that enables citizens to discover relevant government schemes using natural language queries in Kannada.

The system leverages **multilingual sentence embeddings** generated by the `paraphrase-multilingual-MiniLM-L12-v2` transformer model and employs **Cosine Similarity** as the core semantic matching algorithm. User queries are converted into high-dimensional mathematical vectors and compared against pre-computed scheme embeddings stored in a **MongoDB Atlas** cloud database. The system returns the top three most relevant schemes ranked by a computed AI confidence score, subject to a configurable minimum relevance threshold of 40%.

The application is built on a modular **FastAPI** backend and a **React (Vite)** frontend, following industry-standard REST API conventions.

## 2. Introduction

The Government of Karnataka has launched numerous welfare schemes targeting diverse demographics — women, farmers, students, BPL families, and transgender individuals. However, a large proportion of eligible citizens remain unaware of schemes they qualify for, largely because:

- Official information is published in English, which is inaccessible to many Kannada-speaking citizens.
- Search portals require exact keyword matches, failing to understand intent.
- Navigating multiple government websites is time-consuming and confusing.

This project addresses these challenges by building an intelligent assistant that understands the **semantic intent** of queries written in Kannada and matches them to the most contextually relevant government schemes. Unlike traditional keyword-search engines, this system understands that a query like "ಮಹಿಳೆಯರಿಗೆ ಹಣ" (money for women) should return the **Gruha Lakshmi** scheme, even if those exact words do not appear in the scheme description.

## 5. System Architecture

```
+----------------------------------------------------------+
|                     USER (Browser)                       |
|              React Frontend  (Vite / Port 5173)          |
+---------------------------+------------------------------+
                            |  HTTP POST /api/search
                            |  { "text": "..." }
                            v
+----------------------------------------------------------+
|               FastAPI Backend  (Port 8000)               |
|                                                          |
|  +----------+   +--------------+   +-----------------+  |
|  |  Routes  |-->|  NLP Module  |-->|  Search Module  |  |
|  | /search  |   |  utils/nlp   |   |  utils/search   |  |
|  +----------+   +--------------+   +--------+--------+  |
|                                             |            |
+---------------------------------------------+------------+
                                              | Query DB
                                              v
                          +-------------------------------+
                          |     MongoDB Atlas Cloud       |
                          |  Database  : authnlp          |
                          |  Collection: schemes          |
                          |  Fields    : name, desc,      |
                          |              keywords,        |
                          |              embedding        |
                          +-------------------------------+
```

### Data Flow (Step-by-Step):
1. User types a Kannada query into the React search bar.
2. React sends a `POST` request to `http://localhost:8000/api/search` via Axios.
3. FastAPI receives the text and calls the **NLP Module** to clean and embed it.
4. The **Search Module** fetches all scheme embeddings from MongoDB.
5. Cosine Similarity is calculated between the query vector and each scheme vector.
6. Schemes scoring below 40% are discarded. Remaining results are sorted and top 3 are returned.
7. React renders the result cards with scheme details and AI confidence percentages.

## 6. Modules Description

### 6.1 Database Module (`backend/database/database.py`)
Manages the asynchronous connection to MongoDB Atlas using the `motor` driver. The connection URL is loaded securely from a `.env` environment variable file, ensuring credentials are never exposed in source code.

### 6.2 NLP Preprocessing Module (`backend/utils/nlp.py`)
- **`clean_kannada_text(text)`**: Uses Regular Expressions targeting the Unicode Kannada character block (`\u0C80–\u0CFF`) to preserve Kannada script while removing irrelevant punctuation.
- **`tokenize_text(text)`**: Splits cleaned text into individual word tokens using space-based tokenization.
- **`generate_embedding(text)`**: Feeds the cleaned text into the multilingual `SentenceTransformer` model and returns a 384-dimensional float vector.

### 6.3 Search Algorithm Module (`backend/utils/search.py`)
- **`calculate_cosine_similarity(vec1, vec2)`**: Implements the cosine similarity formula from scratch using Python's standard `math` library — no external scientific libraries required.
- **`find_top_matching_schemes(...)`**: Scores all schemes, applies the confidence threshold filter, sorts results, and returns the top N matches.

### 6.4 API Routes (`backend/routes/`)
- **`health.py`**: A `GET /api/health` endpoint for server status verification.
- **`search.py`**: A `POST /api/search` endpoint that accepts text, invokes the NLP and Search modules, and returns structured JSON with the top matched schemes.

### 6.5 Database Seed Script (`backend/seed.py`)
A one-time setup script. Loads the AI model, generates embeddings for 5 sample schemes, and inserts complete documents into MongoDB Atlas.

### 6.6 React Frontend (`frontend/src/`)
- **`App.jsx`**: Core React component managing state (query, results, loading, error).
- **`App.css`**: Dark-mode glassmorphism interface with animations and gradient result cards.
- **`services/api.js`**: Axios service layer abstracting all HTTP communication.

## 7. Algorithms Used

### 7.1 Sentence Embedding Generation

The system uses the **`paraphrase-multilingual-MiniLM-L12-v2`** model from HuggingFace.

| Property | Value |
|---|---|
| Model Type | Transformer (BERT-based) |
| Languages Supported | 50+ (including Kannada) |
| Output Dimensions | 384 |
| Model Size | ~471 MB |
| Parameters | ~118 Million |

The model encodes complete sentence meaning into a fixed-length 384-float vector. It is language-agnostic — the vector for "ಮಹಿಳೆಯರಿಗೆ ಹಣ" (Kannada) and "money for women" (English) will be positioned near each other in the vector space.

### 7.2 Cosine Similarity

Cosine Similarity measures the **angular proximity** between two vectors. It is preferred over Euclidean Distance for text because it is length-insensitive — a short query can match a long description.

**Formula:**
```
              A · B
cos(θ) = ─────────────
            ||A|| × ||B||

Where:
  A · B  = Dot Product = Sum of (Ai × Bi)
  ||A||  = Magnitude of A = sqrt(Sum of Ai^2)
  ||B||  = Magnitude of B = sqrt(Sum of Bi^2)
```

**Score Interpretation:**

| Score | Interpretation |
|---|---|
| 100% | Identical meaning |
| 70–99% | Highly relevant |
| 40–69% | Moderately relevant |
| Below 40% | Not relevant — filtered out |

### 7.3 Confidence Threshold Filtering

A configurable `MIN_CONFIDENCE_THRESHOLD = 40.0` is applied post-ranking. If the highest-ranked scheme does not clear this threshold, an empty result set is returned with the message: *"No relevant government scheme found."* This prevents misleading low-confidence suggestions.

## 9. Future Enhancements

1. **Vector Database Integration:** Replace in-memory cosine similarity with **MongoDB Atlas Vector Search** or **Pinecone** to support thousands of schemes with millisecond response times.

2. **Gemini / OpenAI API Integration:** Replace the locally-hosted MiniLM model with the **Google Gemini Embeddings API** for superior Kannada comprehension without the 471 MB local download.

3. **Eligibility Filtering:** Add structured eligibility fields (gender, BPL status, age, occupation) and implement hard filters before semantic ranking.

4. **Voice Input Support:** Integrate the **Web Speech API** to allow citizens to search by speaking in Kannada, making the app accessible to users with low literacy.

5. **Multi-Language Support:** Extend to support Hindi, Telugu, and Tamil to cover a broader South Indian citizen base.

6. **Admin Dashboard:** Build a secure portal for government officials to add, update, and manage scheme information.

7. **Feedback Loop:** Allow users to rate result relevance and use data to fine-tune the model for the government scheme domain.

8. **Mobile Application:** Port to **React Native** for a standalone Android/iOS app.

## 10. Conclusion

The **Kannada Government Scheme Assistant** successfully demonstrates the practical application of Natural Language Processing to a real-world social problem. By combining **multilingual transformer embeddings** with a mathematically rigorous **Cosine Similarity** ranking engine, the system bridges the linguistic gap between Kannada-speaking citizens and the wealth of government welfare information available to them.

The project follows a clean, modular software architecture, applying industry-standard practices including environment-variable-based secret management, asynchronous database access, RESTful API design, and component-based frontend development. The core similarity algorithm is built from first principles using pure Python — making it both academically understandable and practically functional.

The configurable confidence threshold demonstrates an awareness of real-world constraints: returning no result is always better than returning a misleading one. This principle of **responsible AI output** is a cornerstone of trustworthy systems designed for public use.

This project establishes a solid academic and technical foundation for a production-grade regional-language AI assistant, with clear pathways toward a fully deployed, citizen-facing government service.
